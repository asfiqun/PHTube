Few Things You Need to Know about Javascript

The scope of var, let, and const

In JavaScript, users can declare a variable using 3 keywords that are var, let, and const. The scope of a var variable is functional scope. The scope of a let variable is block scope. The scope of a const variable is block scope. It can be updated and re-declared into the scope.

The use cases of null and undefined

Null and undefined are both values in JavaScript used to represent missing or non-existent data. Undefined means the variable has been declared, but its value has not been assigned. Null means an empty value or a blank value. Some use cases are-

  • Initialization: Variables are often set to undefined when declared and assigned null or undefined when you want to explicitly indicate the absence of a value.
  • Function Parameters: Parameters that are not provided in function calls are often undefined, allowing functions to handle optional arguments.
  • Object Properties: When you want to check if an object has a specific property, you can compare it to undefined or use the "in" operator.
  • Error Handling: Functions may return null or throw an error to indicate exceptional conditions.
  • Default Values: You can use null or undefined to set default values in function arguments or object properties.
  • Data Validation: When validating user inputs or API responses, you might encounter null or undefined values to handle edge cases
